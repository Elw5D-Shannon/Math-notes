# 有限群的结构

## 背景

- **视角变换**：
  - 之前我们看待群时，实际上都是"生成元"视角，重点关注的是它的封闭性
  - 引入群作用后，就为我们提供了全新的视角："运动"视角，重点关注的是它的对称性
- **背景**：
  - 实际上，群论被发明时，主要目的就是研究代数方程根的置换，即群作用。但后面我们知道，群作用只是群的表示，它不能唯一定义一个群，所以群论都是从生成元视角讲起
- **疑问**：为什么有限群的分类一定要依托于群作用？它不只是看待群的视角之一吗？
- **解答**：
  - 首先需要解答根本问题：什么叫有限群的分类？答：希望能按照阶数依次找出哪些群彼此同构，从而将各种群分为抽象的同构类
  - 那么新的问题来了：你说的"各种群"是怎么来的？显然不是用生成元定义的吧？因为根据运算法则可以直接看出是否同构
    - 那你又说：肯定是不给运算法则，要自己找生成元和运算法则，再验证是不是群，然后再看是否同构。好吧好吧，那意思是完全从零开始归纳：首先假设只有 $k$ 个生成元，然后可能的运算法则是……
    - 这样子肯定太蠢了，不过只靠"生成元"视角的话，这个方法确实是唯一的方法。但数学的核心不就是构造系统来简化问题吗？既然"生成元法"已经走到头了，为什么不开拓新的方法呢？
  - 我们回归本源。实际中不少群都是根据应用来命名和发现的，比如置换群就是字典上置换的群，二面体群就是平面正多边形的顶点置换群等。我们发现把字典抽象成集合，把置换抽象成作用，那么就得到了这个应用的数学描述，即群作用（群的置换视角）
    - 显然字典是没有封闭性约束的，直接取就可以。而在群作用视角下，群元素都是某种置换。显然寻找置换的结构和规律比思考运算法则要容易得多，也就达到了我们简化问题的目的
    - PS1：虽然群作用不能唯一定义一个群，但任意群都有作用。所以我们只需消去那些"冗余"的构造即可
    - PS2：群也不只有一种表示方法，矩阵表示也是现代有限群分类的重要工具。抽象代数中的西罗定理思路是"将有限群拆成结构简单的素数幂阶群，再根据置换表示的性质进行分类"。但对于高阶群和有限单群，矩阵表示是必须的（但是很难，所以另开了一门群表示论课程）
    - PS3：其实群作用放到前面讲更有助于理解教材，但是GTM73是布尔巴基风格，目的是训练公理化思维，要讲定义就必须先证明它的合理性，所以即使是生成元视角也等到了自由群章节才将。像Artin这种风格的书就不一样，但相应的问题就是Artin写的乱七八糟。如果要补充动机的话，推荐Jacobson，只不过我是没时间看这种大部头了
- **顺序问题**：
  - 必须强调的是，在历史上是先研究置换群，再提出群的公理化定义。群作用不是专门为了分类而提出的"需要学习的新事物"，而是教材人为地把群的本质之一放到了后面讲。
  - 实际上有了群作用，前面的很多知识都更好理解
